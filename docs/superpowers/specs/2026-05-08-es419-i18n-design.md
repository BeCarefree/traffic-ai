# 西班牙語（es-419）多國語系擴充設計

## 文件資訊
- 日期：2026-05-08
- 專案：`traffic-ai/web`
- 主題：在既有繁中/英文架構上新增拉丁美洲西語（`es-419`）
- 狀態：已完成設計、待使用者審閱

## 目標與範圍
本設計目標是在現有網站多國語系中新增 `es-419`，並維持既有語言切換與儲存體驗。

本次範圍限定為：
- 僅翻譯 `src/i18n/translations.ts` 目前已存在的字串 key。
- 右上角語系切換從 2 個按鈕改為 3 個按鈕（`中` / `EN` / `ES`）。
- 西語缺字時採用 fallback 到英文；英文也缺時 fallback 到繁中原文。

本次不包含：
- 新增 `translations.ts` 以外的新翻譯 key。
- 全站硬編碼文字盤點與重構。
- 文案審校流程（僅先以可用翻譯為目標）。

## 需求定案（與使用者確認）
- 西語版本：`es-419`（拉丁美洲通用）。
- 翻譯範圍：選項 A（僅 `translations.ts` 現有字串）。
- 缺字 fallback：選項 A（先英文、再繁中）。
- UI 行為：語系切換為三按鈕樣式。

## 設計方案比較與採用
### 方案 1（採用）
將翻譯資料改為「單一 key 對應多語值」結構，集中管理 `en` 與 `es419`。

優點：
- 擴充新語言時不需再複製多份平行字典。
- fallback 邏輯可在 `translate()` 一處統一定義。
- 型別更容易限制語言代碼範圍。

成本：
- `translations.ts` 需要一次性重構資料形狀。

### 方案 2（未採用）
維持 `zh -> en` 字典，另外新增 `zh -> es419` 字典。

未採用原因：
- 短期改動小，但長期維護會分散、重複邏輯增多。

### 方案 3（未採用）
拆分多檔語系（`zh.ts` / `en.ts` / `es419.ts`）與共用 key。

未採用原因：
- 對目前專案規模偏重，調整面太大。

## 架構與元件設計
### 1) 型別與翻譯資料（`src/i18n/translations.ts`）
- `Lang` 由 `'zh' | 'en'` 擴充為 `'zh' | 'en' | 'es419'`。
- 翻譯資料改為：
  - key：繁中字串（既有模式不變）
  - value：語系對照物件（至少可含 `en`、`es419`）
- 建議型別：
  - `type NonZhLang = 'en' | 'es419'`
  - `Record<string, Partial<Record<NonZhLang, string>>>`

### 2) 語言切換與持久化（`src/i18n/LanguageProvider.tsx`）
- `readInitialLang()` 支援讀取 `es419`。
- `document.documentElement.lang` 設定規則：
  - `zh` -> `zh-Hant`
  - `en` -> `en`
  - `es419` -> `es-419`
- `localStorage` key 維持 `traffic-ai.lang`，相容既有使用者資料。

### 3) 右上角語言切換 UI（`src/App.tsx`）
- `LanguageToggle` 新增第三顆 `ES` 按鈕。
- Active 狀態判斷新增 `lang === 'es419'`。
- 點擊行為：`setLang('es419')`。

## 資料流與轉譯規則
### `translate(text, lang)` 期望行為
1. `lang === 'zh'`：直接回傳原始 `text`（繁中 source-of-truth）。
2. `lang === 'en'`：
   - 若存在英文翻譯，回傳英文。
   - 若不存在，回傳原始繁中。
3. `lang === 'es419'`：
   - 若存在西語翻譯，回傳西語。
   - 若西語缺字但英文存在，回傳英文。
   - 若西語與英文皆缺，回傳原始繁中。

## 錯誤處理與相容策略
- 讀取到未知語言值（localStorage 汙染）時，回退到 `zh`。
- 翻譯字典缺 key 或缺語言值時，不拋錯，依 fallback 規則穩定顯示。
- 保留既有繁中 key 作為主鍵，避免頁面顯示 `undefined`。

## 驗證與測試計畫
### 手動驗證
- 切換 `中` / `EN` / `ES`，確認 UI 即時更新。
- 重新整理頁面後，語言選擇可被記住（localStorage 生效）。
- 檢查 `<html lang>` 對應值為 `zh-Hant`、`en`、`es-419`。
- 任選 1~2 個 key 暫時移除 `es419` 值，確認會回退英文。
- 再移除同 key 的英文值，確認最終回退繁中。

### 回歸風險檢查
- 確認使用 `t()` 的主要頁面（Dashboard、Devices、Incident、CMS、Data Monitor）無 runtime 錯誤。
- 確認語言切換按鈕樣式在三按鈕情境下仍可讀且可點擊。

## 實作邊界
- 不調整翻譯 key 命名策略（維持目前以繁中文字串作 key）。
- 不在此階段導入 ICU、命名 key（如 `dashboard.title`）等 i18n 重構。
- 不擴充自動化翻譯工具鏈。
